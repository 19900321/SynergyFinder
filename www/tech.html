1. Reference models
The synergistic effect can be determined as the excess of observed effect over expected effect calculated with
reference models (synergy scoring models). All of the models make different assumptions regarding the expected effect.
Currently, four reference models are available in SynergyFinder: HSA, Bliss, Loewe and ZIP.
1.1. HSA
HSA model is one of the simplest reference models, which states that the expected combination effect is simply
the maximum of the single drug responses at corresponding concentrations.
1.2. Bliss

independently, and the expected combination effect (I

1.3. Loewe
Loewe additivity model calculates the expected response as if both drugs are the same. Hence, a slice through
the response surface at any chosen effect should show a linear relationship between the doses of the two drugs. For
example, if 50% inhibition is achieved separately by 1 μM of drug A or 2 μM of drug B, a combination of 0.5 μM of A
and 1 μM of B should also inhibit by 50%. Formally, the expected response at combined concentrations X, Y is the

1.4. ZIP
Zero interaction potency (ZIP) model captures the drug interaction relationships by comparing the change in the
potency (effect at certain dose level) of the dose–response curves between individual drugs and their combinations. It
assumes that two non-interacting drugs are expected to incur minimal changes in their dose–response curves. The
formulation of the model can be found at https://www.ncbi.nlm.nih.gov/pmc/articles/PMC4759128/.
The ZIP model takes the advantages of both the Loewe additivity and the Bliss independence models, aiming at
a systematic assessment of various types of drug interactions patterns that may arise in a high-throughput drug
combination screening.

Data analysis
3. Input data
For easier user experience, two possible input file formats (Table and Matrix) are allowed in SynergyFinder. Therefore,
the same input data can be provided in the two alternative formats.
3.1. Table format
In the Table format, the input dose-response matrix is represented as a table where each row contains the
information about the one cell in the dose-response matrix (Fig. 1).
The input table must comprise the following columns:
1. PairIndex - index of drug-pair to be analyzed
2. Response - %inhibition or %viability values
3. Drug1 – name of the first drug
4. Drug2 – name of the second drug
5. Conc1 - concentration of Drug1
6. Conc2 - concentration of Drug2
7. ConcUnit - unit of concentration
The orders of rows and columns are arbitrary. The number of drug combinations (PairIndex) provided in the input
file is unlimited. The only restriction is imposed on the size of a dose-response matrix, which should comprise at least
three rows and columns, so that sensible synergy scores can be calculated.
The expected file extensions are either *.xlsx, comma-delimited *.csv, or comma-delimited *.txt file (see example
data).
Fig. 1. An input file in Table format.
3.2. Matrix format
In the Matrix format, the dose-response matrix is represented in a matrix form with drug concentrations shown
along the top and left side of the matrix (Fig. 2).
The three rows should precede each dose-response matrix:
1. Drug1: name of the first drug
2. Drug2: name of the second drug
3. ConcUnit: unit of concentration
The drug concentrations should be located at the top and left side of the matrix, where left side concentrations
correspond to Drug1 and concentrations shown at the top correspond to the Drug2. The number of drug combinations
provided in the input file is unlimited. The only restriction is imposed on the dose-response matrix size, which should
comprise at least three rows and columns, so that sensible synergy scores can be calculated.
The expected file extensions are either *.xlsx, comma-delimited *.csv, or comma-delimited *.txt file. (see example
data)
Fig. 2. An input file in Matrix format. Two dose-response matrices (for AT-406 & Navitoclax and NVP-LGK974 &
Alpelisib drug combinations) are provided as an example.
4. Data analysis
After uploading the input file the user needs to specify the type of phenotypic response (either %inhibition or
%viability). In case of %viability response type, the provided %viability values will be converted to %inhibition by the
formula: %inhibition = 100 - % viability.
After specifying the type of phenotypic response the user may firstly obtain an overview of the provided data
(full dose-response matrix, as well as the plots of phenotypic responses for the single drugs) to check for the
problematic data points or go directly to the calculation of synergy scores.
4.1. Visualization of input dose-response data
Visualization of input dose-response data is done by switching on the corresponding toggle button (use the
interactive web tour for step-by-step explanation). It will give the user an overview of the full dose-response matrix, as
well as the plots of phenotypic responses for the single drugs (i.e. the first column and first row in the dose-response
matrix) (Fig. 3), fitted by four-parameter logistic curves. The visualization enables the user to identify any problematic
data points, such as negative phenotypic responses or outliers for which a dose-response curve cannot be properly fitted.
The ‘Correction’ option can be switched on to apply a base line correction method on the single drug dose-responses to
minimize the impact of such problematic data points in the subsequent synergy scoring steps.
Fig. 3. Visualization of input dose-response data
In case there is a missing value in the dose-response matrix it will be colored in gray and a synergy score will
not be calculated for this drug combination (Fig. 4A). SynergyFinder allows for correcting these cells with missing
values (Fig. 4A) or outliers (Fig. 4B). The correction is done using the ‘Estimate missing responses or outliers’ widget.
The missing responses or outliers will be estimated by taking the average of the nearest responses of both drugs. For
example, the missing value from the response matrix in Fig. 4A will be estimated as (70.03+70.28+73.47+73.11)/4
=71.72

4.2. Calculation and visualization of synergy scores
Before calculations of the synergy scores two parameters should be specified:
1. Method - a reference model by means of which the expected effect (response) will be calculated (HSA,
Loewe, Bliss or ZIP).
2. Correction - a baseline correction (ON/OFF), trying to fix negative or irregular responses.
After these parameters have been specified the calculation of synergy scores and visualization of synergy maps is
done by switching on the corresponding toggle button (more details in the web tour).
The synergy score for a drug combination is averaged over all the dose combination cells. The 2D and 3D synergy maps
highlight synergistic and antagonistic dose regions in red and green colors, respectively (Fig. 5).
SynergyFinder also helps to find the most synergistic area in the synergy interaction map. In a 3D synergy map
users are provided with possibilities to (i) rotate, (ii) zoom, (iii) hover over to show synergy scores for any part of the
synergy map and (iv) save the resulting image as HTML, PNG or SVG. In a 2D synergy map it also allows zooming in
specific regions by brushing and double-clicking. After zooming the synergy score will be recalculated for the chosen
area.
Fig 5. Visualization of synergy maps.
4.3. Report generation
SynergyFinder allows saving all the results in one PDF file. Three types of reports are available: static report,
dynamic report and short report (Fig. 6). For a static report, a user must specify which types of plots and which drug
combinations should be included in the report (by default, all plots for all drug pair are included). In order to exclude a
certain drug pair, click on it from the ‘Choose drug pairs’ box and then press DEL button. A dynamic PDF report allows
the rotation of 3D synergy maps, in order to find the best view angle for printing. A short report is the concise version,
where only dose-response matrices and 2D synergy maps are included. A user may also specify whether the most
synergistic areas should be shown on the synergy maps. The synergy scores for all the drug combinations may be
downloaded in a separate table for further analyses.
